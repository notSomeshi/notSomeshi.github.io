OpenRouter 上的权力转移:中国开源模型如何吃下一半流量

#开源模型 #OpenRouter #市场份额

如果说各家基准测试榜单衡量的是”模型有多聪明”,那 OpenRouter 的用量榜衡量的是一件更残酷的事:开发者真正把钱花给了谁。而 2026 年年中的这份账单,记录了一场悄无声息的权力转移。

一、一年之间:70% → 30%

OpenRouter 是目前最大的多模型 API 聚合网关,它的 token 流量分布被广泛当作”AI 应用层真实需求”的代理指标。2025 年 6 月,Google、OpenAI、Anthropic 三家美国厂商合计占其约 70% 的 token 份额;到 2026 年 6 月,**这个数字跌到了约 30%**。Anthropic 的份额从峰值腰斩至 13.3%,即便如此仍是最大的单一闭源阵营。

填补真空的是中国开源权重模型:DeepSeek 单家占全平台 token 量的 16.3%,超过 Google、Anthropic、OpenAI 任何一家;按模型算,DeepSeek V4 Flash 高居用量榜第一,腾讯 Hy3 Preview 第二,用量前十里中国开源模型占六席;DeepSeek、小米、腾讯、MiniMax、Qwen 合计约占前十流量的 44%。智谱的 GLM-5.2 则被评论者称为”迷你 DeepSeek 时刻”——发布后流量单月暴涨近十倍。

二、流量与收入:两个平行的榜单

但在为”开源已胜”下结论之前,要看第二份账单。OpenRouter 的数据同时显示:Anthropic 以约 12.3% 的 token 量,拿走了远超比例的收入——因为高端闭源模型的单 token 价格是开源模型的许多倍。用量最高的 Claude 模型 Opus 4.7 仍稳居全平台第三,是排名最高的闭源模型。

换句话说,市场正在清晰地分层:

  • 批量层:翻译、摘要、分类、常规代码补全——对智能上限不敏感、对价格极度敏感,被便宜到近乎免费的中国开源模型整体接管;
  • 攻坚层:复杂 Agent 任务、生产级代码交付、高风险决策——买家愿意为每个 token 多付十倍,闭源旗舰(参见本站模型天梯的能力榜)守住了这一层的定价权。

这正是经典的”性能过剩”市场结构:当第二梯队的能力越过”够用线”,第一梯队就只能退守利润最厚、也最窄的顶端。

三、对开发者与独立开发者意味着什么

  1. 默认技术栈正在改写。一年前”默认 GPT/Claude、特殊需求再找便宜替代”的习惯,正在反转为”默认开源便宜模型、攻坚任务再上旗舰”。混合路由(model routing)从优化技巧变成了标配架构。
  2. 价格地板还在下沉。GLM-5.2、DeepSeek V4 系列用接近成本的价格供应 80 分能力,任何依赖”转售模型能力”的产品,毛利都在被重新定价。
  3. 能力榜与用量榜要一起看。这也是本站模型天梯新增「社区体感」维度(Arena 众测投票 + OpenRouter 用量份额)的原因:基准分数告诉你模型能做什么,真实流量告诉你市场认为它值多少钱——两者的落差里,藏着这个行业的全部商业故事。
